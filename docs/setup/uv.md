# UV 설치 및 초기화

uv는 Python 패키지 관리와 가상환경을 빠르게 다루는 도구입니다.

---

## 가상환경, 왜 필요할까요? 🤔

파이썬 개발에서 **가상환경(Virtual Environment)** 설정은 선택이 아닌 필수입니다. 처음 시작할 때 왜 이런 번거로운 과정이 필요한지 이해하는 것이 중요합니다.

### 1. 의존성(Dependency)의 개념
우리가 프로그램을 만들 때 직접 모든 코드를 짜지 않고, 남들이 만들어둔 '라이브러리(패키지)'를 가져다 씁니다. 이때 우리 프로그램이 제대로 작동하기 위해 **의존(의지)**하고 있는 외부 코드를 **의존성**이라고 부릅니다.

### 2. 의존성 충돌 (Dependency Hell) 💥
문제는 프로젝트가 여러 개로 늘어날 때 발생합니다.

*   **프로젝트 A**: 구형 라이브러리 `X` (버전 1.0) 필요
*   **프로젝트 B**: 최신 기능을 위한 라이브러리 `X` (버전 2.0) 필요

만약 내 컴퓨터 전체에 라이브러리 하나만 설치할 수 있다면 어떻게 될까요? A를 돌리려면 버전을 낮춰야 하고, B를 돌리려면 다시 높여야 합니다. 프로젝트가 10개로 늘어나고 라이브러리가 100개가 되면, 버전 맞추느라 정작 개발은 시작도 못 할 수 있습니다.

### 3. 가상환경: 프로젝트별 독립된 작업실 🏠
가상환경은 각 프로젝트만을 위한 **공구함(또는 작업실)**을 따로 만드는 것과 같습니다.
- **프로젝트 A 전용 공구함**: 1.0 버전 도구들 보관
- **프로젝트 B 전용 공구함**: 2.0 버전 도구들 보관

이렇게 하면 서로의 환경을 침범하지 않고, 언제든지 안전하게 프로젝트를 실행할 수 있습니다. `uv`는 바로 이 가상환경을 세상에서 가장 빠르게 만들어주고 관리해 주는 도구입니다.

## 프로젝트 초기 설정

다음 흐름을 기본으로 합니다.

1. 저장소 생성
2. uv로 가상환경 및 의존성 설정
3. 실행 방법 문서화

### ⚠️ 중요: 프로젝트 폴더 및 이름 정하기 규칙
프로젝트를 시작할 때 폴더 이름을 대충 지으면 나중에 해결하기 힘든 오류를 만날 수 있습니다. 다음 규칙을 꼭 지켜주세요!

1.  **반드시 영어 이름을 사용하세요**: 한글이나 특수문자가 들어간 폴더명은 파이썬 내부 라이브러리에서 인식을 못 하거나 경로(Path) 오류를 일으킬 수 있습니다 (예: `나의-프로젝트` (X) -> `my-study-project` (O)).
2.  **라이브러리 이름과 겹치지 않게 하세요**: 만약 프로젝트 폴더 이름을 `requests`나 `fastapi`처럼 유명한 라이브러리 이름과 똑같이 지으면, `import requests`를 했을 때 진짜 라이브러리 대신 내 프로젝트를 불러오려고 시도하다가 오류가 납니다.
3.  **소문자와 하이픈(-)을 권장합니다**: 공백(Space) 대신 하이픈이나 언더바(_)를 사용하세요.

### 예시

```bash
# 1) 새 폴더
mkdir my-study-project
cd my-study-project

# 2) uv로 가상환경 만들기 (버전 지정 가능)
uv venv --python 3.11

# 3) 의존성 추가
uv add fastapi

# 4) 실행
uv run python -m pip list
```

## venv에서 Python 버전 지정

```bash
uv venv --python 3.11
uv venv --python 3.12
```

## 일반적인 실행 방법

```bash
# 가상환경 실행
uv run python app.py

# 패키지 모듈 실행
uv run pytest
```

## uv pip install vs uv add 차이

- `uv pip install`: pip 호환 방식으로 즉시 설치만 수행 (프로젝트 메타데이터 업데이트 없음)
- `uv add`: 프로젝트 의존성으로 기록하고 설치까지 수행 (권장)

예시:

```bash
uv pip install requests
uv add requests
```

## uv init

새 Python 프로젝트 기본 구조를 빠르게 생성합니다. 현재 폴더 이름이 프로젝트 이름으로 자동 지정됩니다.

```bash
uv init
```

### 프로젝트 이름 별도로 지정하기
만약 폴더 이름과 상관없이 실제 프로젝트(패키지)의 이름을 따로 정하고 싶다면 `--name` 옵션을 사용하세요.

```bash
# 폴더는 study-01 이지만, 프로젝트 이름은 my_app으로 설정
uv init --name my_app
```
